Walk through the practice exam or problem set: $ARGUMENTS

## Setup
1. Load raw/<course>/<exam-file>
2. Read wiki/gaps.md to know current weak spots

## For Each Problem
1. Present the problem clearly
2. Wait for the student to attempt it
3. After their answer, regardless of correctness, give full explanation:
   - Core concept — plain English, analogy first
   - Step-by-step reasoning — why each step follows
   - Full derivation/calculation — every step shown
   - Real-world connection
   - Visual SVG/HTML diagram if the concept is spatial/mechanistic
4. If they struggled → add/update a gap entry in wiki/gaps.md

## After All Problems
- Summary: score (X/Y), top concepts that need work
- Update wiki/gaps.md with any new gaps found
- Append to wiki/log.md:
  ```
  ## [YYYY-MM-DD] practice | <filename> | Score: X/Y | Gaps found: <list>
  ```
- Suggest follow-up: specific /quiz targets based on what was missed
